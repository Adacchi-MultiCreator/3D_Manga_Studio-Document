    翻訳について

    「3D Manga Studios」アドオンは、Blenderの組み込み翻訳システムを活用して多言語対応を実現しています。これにより、BlenderのUI言語設定に応じて、アドオンのインターフェースや説明文が自動的に翻訳されます。

    1. 翻訳辞書の構造

    翻訳データは、translation/ディレクトリ内のJSONファイルに格納されています。現在、以下の2つのファイルがあります。

    • UI words.json: アドオンのユーザーインターフェース（パネル名、ボタンテキスト、ツールチップなど）で使用される一般的な単語やフレーズの翻訳を定義します。
    • Manga Tone Shader parameter.json: マンガトーンシェーダーのノードグループの入力ソケット名や説明文など、シェーダー固有のパラメータの翻訳を定義します。

    これらのJSONファイルは、以下のような構造を持っています。

    [
  {
    "Key": [
      "English Key 1",
      "English Key 2"
    ]
  },
  {
    "English Text": [
      "English Text 1",
      "English Text 2"
    ]
  },
  {
    "Japanese Translation": [
      "日本語訳 1",
      "日本語訳 2"
    ]
  }
]
    • Key: 翻訳のキーとして使用される英語の文字列のリストです。これは、コード内で翻訳関数に渡される元の文字列と一致する必要があります。
    • English Text: Keyに対応する英語のテキストのリストです。BlenderのUI翻訳がオフの場合や、英語が選択されている場合に表示されます。
    • Japanese Translation: Keyに対応する日本語の翻訳テキストのリストです。BlenderのUI言語が日本語に設定されている場合に表示されます。

    2. 翻訳辞書のロードと登録

    アドオンがBlenderにロードされる際、__init__.py内のload_translation_dict()関数がこれらのJSONファイルを読み込み、Blenderの翻訳システムが使用できる形式の辞書を作成します。

    # __init__.py 内の抜粋

def load_translation_dict():
    current_dir = os.path.dirname(__file__)
    translation_dir = os.path.join(current_dir, "translation")
    ui_words_path = os.path.join(translation_dir, "UI words.json")
    shader_params_path = os.path.join(translation_dir, "Manga Tone Shader parameter.json")

    blender_translation_dict = {
        "en_US": {},
        "ja_JP": {}
    }

    # UI words.json の読み込み処理...
    # Manga Tone Shader parameter.json の読み込み処理...

    return blender_translation_dict

# register() 関数内で翻訳辞書を登録
def register():
    global global_translation_dict
    global_translation_dict = load_translation_dict()
    bpy.app.translations.register(__name__, global_translation_dict)
    # ...

    bpy.app.translations.register(__name__, global_translation_dict)によって、作成された翻訳辞書がBlenderの翻訳システムに登録されます。

    3. コード内での翻訳関数の使い方

    アドオンのPythonコード内でテキストを翻訳するには、Blenderの翻訳ヘルパー関数を使用します。

    • bpy.app.translations.pgettext_iface(text): 主にUI要素（パネルのラベル、ボタンのテキスト、プロパティ名など）の翻訳に使用されます。Blenderのユーザー設定で「UI翻訳」が有効になっている場合にのみ翻訳が適用されます。
    • bpy.app.translations.pgettext(text): 一般的なテキストやログメッセージなど、UI翻訳設定に依存しない翻訳に使用されます。

    さらに、このアドオンでは、shader_operator.py内でカスタムの翻訳ヘルパー関数_()とT()が定義されています。これらはpgettext_ifaceをラップしており、BlenderのUI翻訳設定がオフの場合には元の英語テキストを返すように設計されています。

    # shader_operator.py 内の抜粋

def _(text: str) -> str:
    prefs = bpy.context.preferences.view
    if not prefs.use_translate_interface:
        return text
    return bpy.app.translations.pgettext_iface(text)

# 使用例
layout.label(text=_("Light Collection Management"), icon=\'LIGHT\')

    4. 新しい翻訳を追加する方法

    アドオンに新しいUI要素やシェーダーパラメータを追加し、それを多言語対応させるには、以下の手順に従います。

    1. 翻訳キーの決定: 新しく追加するテキストに対応する、一意で分かりやすい英語のキーを決定します。このキーは、コード内で翻訳関数に渡す元の文字列となります。
    2. JSONファイルの更新: translation/UI words.jsonまたはtranslation/Manga Tone Shader parameter.jsonの適切な方に、新しいキー、英語のテキスト、および対応する日本語の翻訳を追加します。各配列の同じインデックスに新しいエントリを追加するように注意してください。
      • 例: UI words.jsonに新しいボタンのテキストを追加する場合 json [ { "Key": [ // ... 既存のキー "New Button Text" ] }, { "English Text": [ // ... 既存の英語テキスト "New Button Text" ] }, { "Japanese Translation": [ // ... 既存の日本語翻訳 "新しいボタンのテキスト" ] } ]
    3. コード内での翻訳関数の使用: アドオンのPythonコード内で、新しいテキストを表示する箇所で_("New Button Text")のように翻訳ヘルパー関数を使用します。

    これらの手順に従うことで、アドオンの多言語対応を維持し、新しいテキストがBlenderのUI言語設定に応じて適切に表示されるようになります。
